What is a Signal?

The Broadest Definition

A signal is any quantity that varies with one or more independent variables and carries information. Sound pressure varying over time, pixel intensity varying across an image, stock prices varying across trading days — all signals. DSP concerns itself primarily with signals that are functions of discrete time: sequences of numbers.

Think of a signal like a long receipt from a cash register. Each line is a measurement taken at a specific moment. The receipt doesn't care about the time between measurements — it only records the values, in order. That ordered sequence of numbers IS your discrete-time signal.

Continuous vs. Discrete

Continuous-Time Signals

A continuous-time signal x(t) is defined for every real value of t. The voltage across an analogue sensor, the pressure wave of a violin note, the temperature in a room — these are all continuous. Mathematically, the domain is the real line.

Discrete-Time Signals

A discrete-time signal x[n] is defined only at integer values of n. It's a sequence: ..., x[-1], x[0], x[1], x[2], ... We use square brackets to distinguish from continuous signals. These arise naturally when we sample a continuous signal at regular intervals, but they can also be generated computationally — no physical sensor required.

x[n] = x_a(nT),   where T = 1/fs is the sampling period Sampling a continuous signal x_a at rate f_s

Energy and Power

E = ∑n=-∞∞ |x[n]|² Total energy of a discrete-time signal
P = limN→∞ (1/(2N+1)) ∑n=-NN |x[n]|² Average power of a discrete-time signal

Energy signals are like a single clap — all the energy arrives in a burst and then it's gone. You can sum it up to a finite number. Power signals are like a humming refrigerator — the total energy is infinite (it never stops), but the average rate of energy delivery is finite and measurable. A pure sine wave is a power signal; a single pulse is an energy signal.

What is a System?

Input-Output Machines

A system T{·} takes an input signal x[n] and produces an output signal y[n]. That's it — a rule mapping input sequences to output sequences. An echo effect is a system. A moving average is a system. Your ear is a system. The key question in DSP is: which systems can we analyse mathematically, implement efficiently, and trust to behave predictably?

y[n] = T{x[n]} System transforms input to output

The Five Key Properties

1. Linearity

A system is linear if it obeys superposition: T{ax₁[n] + bx₂[n]} = aT{x₁[n]} + bT{x₂[n]}. Doubling the input doubles the output. Adding two inputs produces the sum of their individual outputs. This is the single most powerful property in DSP — it lets us decompose complex signals into simple parts, process each part, and recombine.

2. Time-Invariance

A system is time-invariant if delaying the input simply delays the output by the same amount: if y[n] = T{x[n]}, then y[n-k] = T{x[n-k]}. The system's behaviour doesn't change whether you feed it a signal today or tomorrow. Together with linearity, this gives us LTI systems — the bedrock of classical DSP.

3. Causality

A causal system's output at time n depends only on present and past inputs: x[n], x[n-1], x[n-2], ... — never on future values. All real-time physical systems are causal (you can't react to something that hasn't happened). But offline processing can be non-causal — a video editor can look ahead.

4. Stability (BIBO)

Bounded-Input Bounded-Output stability: if every bounded input produces a bounded output, the system is stable. A system that amplifies without limit — where a finite input can produce an infinite output — is unstable. For LTI systems, BIBO stability requires that the impulse response h[n] is absolutely summable: ∑|h[n]| < ∞.

5. Memory

A memoryless system's output depends only on the current input: y[n] = f(x[n]). A system with memory looks at past (or future) inputs. A gain control is memoryless. A delay, an accumulator, or any filter has memory.

An LTI system is like a perfectly consistent factory machine: it does exactly the same thing to every product (time-invariant), it handles mixed batches by processing each item independently and combining results (linear), and you can fully characterise it by feeding in one single test piece and observing the result — the impulse response.

The Impulse Response

The System's DNA

For any LTI system, if you feed in the unit impulse δ[n] (which equals 1 at n=0 and 0 everywhere else), the output h[n] = T{δ[n]} completely characterises the system. Once you know h[n], you can compute the output for ANY input via convolution. This is why LTI systems are so powerful — one experiment tells you everything.

δ[n] = { 1, n=0;   0, n≠0 } Unit impulse (Kronecker delta)
y[n] = x[n] * h[n] = ∑k=-∞∞ x[k] · h[n-k] Output = Input convolved with impulse response

Elementary Signals

Building Blocks

A small set of elementary signals serves as the alphabet of DSP:

  • Unit impulse δ[n]: The "tap" — tests systems, builds other signals
  • Unit step u[n]: 0 for n<0, 1 for n≥0 — models turning something on
  • Complex exponential ejωn: The eigenfunction of LTI systems — goes in as a sinusoid, comes out as a sinusoid (only amplitude and phase change)
  • Real sinusoid cos(ωn + φ): The building block of Fourier analysis
  • Exponential decay anu[n]: Models damping, natural system responses
ejωn = cos(ωn) + j·sin(ωn) Euler's formula — the bridge between exponentials and sinusoids

Why LTI Systems Dominate DSP

The Power of Superposition

Because LTI systems are linear, we can express any input as a sum of weighted, shifted impulses, process each impulse independently (getting a shifted, weighted copy of h[n]), and sum the results. That sum IS convolution. And because they're time-invariant, we can analyse them in the frequency domain — where convolution becomes simple multiplication. This dual representation (time ↔ frequency) is the fundamental insight that makes DSP tractable.

Classical DSP theory assumes LTI systems, but the real world is filled with non-linear, time-varying systems: the human vocal tract changes shape during speech, wireless channels fade and shift, and neural networks are entirely non-linear. Modern DSP increasingly deals with adaptive and non-linear systems — Volterra series, neural network-based filters, and time-varying channel estimators. The question is whether there exists a unified mathematical framework as elegant as LTI theory for these broader system classes, or whether each non-linearity demands its own toolkit.

Active research in non-linear signal processing continues to yield new mathematical tools (kernel methods, tensor decompositions), but no single framework has achieved the universality of LTI theory.

  • Is there a natural "impulse response equivalent" for non-linear systems that provides the same completeness as h[n] does for LTI?
  • How should DSP curricula evolve to balance classical linear theory with the increasingly non-linear reality of modern applications?
